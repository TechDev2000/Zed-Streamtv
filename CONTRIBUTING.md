# Contributing to CricHD

Thank you for your interest in contributing! This document provides guidelines for contributing to the project.

## How to Contribute

### Reporting Bugs
- Use the issue tracker to report bugs
- Describe the bug and include specific steps to reproduce
- Include browser and PHP version information
- Add screenshots if applicable

### Suggesting Features
- Open an issue with the `enhancement` label
- Describe the feature and its use case
- Explain why it would be useful

### Pull Requests
1. Fork the repository
2. Create a new branch from `main`
3. Make your changes
4. Test thoroughly
5. Submit a pull request with a clear description

## Code Style

### PHP
- Follow PSR-12 coding standards
- Use meaningful variable names
- Add comments for complex logic

### JavaScript
- Use ES6+ features where supported
- Comment complex player configurations
- Maintain consistent indentation (2 spaces)

### CSS
- Use BEM naming convention for classes
- Group related properties
- Prefer CSS variables for colors

## Commit Messages
- Use present tense ("Add feature" not "Added feature")
- Use imperative mood ("Move cursor to..." not "Moves cursor to...")
- Limit first line to 72 characters
- Reference issues where applicable

## Testing
- Test on multiple browsers (Chrome, Firefox, Safari, Edge)
- Test responsive design on mobile devices
- Verify HLS streams load correctly

## Questions?
Feel free to open an issue for any questions.
